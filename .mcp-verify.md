## ðŸ”§ Troubleshooting Guide

### Issue: "SUPABASE_SERVICE_ROLE_KEY not found"

**Checks:**
1. Is the environment variable set?
   ```powershell
   [System.Environment]::GetEnvironmentVariable('SUPABASE_SERVICE_ROLE_KEY', 'User')
   ```

2. Did you restart Cursor after setting it?

3. Is the variable name exactly `SUPABASE_SERVICE_ROLE_KEY`?

**Fix:**
```powershell
# Set permanently (Windows)
[System.Environment]::SetEnvironmentVariable('SUPABASE_SERVICE_ROLE_KEY', 'your-key-here', 'User')

# Restart Cursor completely
```

### Issue: "Cannot connect to Supabase"

**Checks:**
1. Is your Supabase project active?
2. Is the URL correct in `.mcp.json`?
3. Is the service role key valid?

**Fix:**
- Verify URL: https://ibqgwakjmsnjtvwpkdns.supabase.co
- Regenerate service role key if needed
- Check Supabase project status

### Issue: "MCP server not starting"

**Checks:**
1. Can you run the server manually?
   ```powershell
   $env:SUPABASE_URL = "https://ibqgwakjmsnjtvwpkdns.supabase.co"
   $env:SUPABASE_SERVICE_ROLE_KEY = "your-key"
   npx -y @modelcontextprotocol/server-supabase
   ```

2. Are there any error messages in Cursor's output?

**Fix:**
- Update Node.js to latest LTS version
- Clear npx cache: `npx clear-npx-cache`
- Check Cursor logs for specific errors

### Issue: "Permission denied" errors

**Checks:**
1. Are you using the service_role key (not anon key)?
2. Does your Supabase plan support the operation?
3. Are RLS policies blocking the operation?

**Fix:**
- Verify you copied the correct key from Dashboard
- Check RLS policies in Supabase Dashboard
- Ensure project is on appropriate plan
